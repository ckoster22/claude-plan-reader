# Sub-Plan 02: Physics Core — Gravity, Collision Regimes, Soft-Body & Growth

> Part of the **Chompy Asteroids** master plan. This sub-plan rebuilds the prototype's bespoke
> simulation as **pure, unit-testable modules** under `src/Operations/`. Rendering, input, world
> spawn/cull, HUD, and **applying** the effect descriptor are later sub-plans and explicit
> non-goals. Operations computes; Workflows (Sub-Plan 03) applies.

## Context

Sub-Plan 01 delivered the scaffold + render spine: a frozen `Body` contract
(`src/domain/Body.js`, `makeBody(opts)`, derived radius via `MASS_TO_R`, `fin` guard,
`_resetIdCounter`), render-only constants in `src/config.js`, a dt-normalized `GameLoop` with an
injected no-op `step(dtNorm)` seam, and a renderer that already consumes `def/dvel/dax/day/flash`
and exposes empty `particles`/`shocks` FX arrays. `src/Operations/` is empty.

The prototype (`.plan-tree/prototype/index.html`, the executable spec) implements all physics
inline in one mutating `step()` loop (lines ~222–449) over a shared `asts` array. This sub-plan
**inverts** that into per-concern PURE functions that return descriptors instead of mutating
globals — the seam the master plan's frozen contracts mandate.

**Two deliberate divergences from the prototype** (decided in the master plan, do NOT silently
port the prototype's version):
1. **Gravity is normalized-mass exponential** `force ∝ (m/M_REF)^GRAV_EXP` with `GRAV_EXP > 1`
   (prototype ships linear `GRAV_EXP=1.0`).
2. **Slow-merge momentum is conserved exactly**: survivor `v = combinedMomentum / newMass`
   (prototype divides by `combinedMass`, which the master plan calls a bug).

## Resolved design decisions (from recon)

- **Purity model:** every Operations function is pure — it **returns** new values/descriptors and
  never mutates its `Body` inputs. The caller (Sub-Plan 03 Workflow) applies them. This keeps
  every function unit-testable in isolation and is required because `Operations/` must not own
  state, counters, or FX.
- **Injectable RNG:** the prototype uses `Math.random()` directly (dent vertex pick, fracture
  spray angles, coincident-center jitter). Thread an `rng = Math.random` parameter through
  `resolveCollision` (and dent/fracture helpers) so fragment geometry, dents, and angles are
  deterministic in tests. Combined with `Body._resetIdCounter()`, fragment ids are reproducible.
- **Regime selection seam (closing-vs-separating made explicit):** the regime is chosen by the
  **magnitude** of the normal component of relative velocity `vn = |relVel · n̂|` against
  `V_MERGE` / `V_FRACTURE`, where closing sign is `s = sign(relVel · n̂)` (`< 0` closing,
  `≥ 0` separating). Explicit, complete decision table (no ambiguous fall-through), chosen to
  prevent the **fracture snap-back** failure mode the DA flagged (freshly-spawned overlapping
  fragments drifting apart at `vn < V_MERGE` would otherwise instantly re-merge, defeating the
  fracture):
  - SLOW (`vn < V_MERGE`): **merge only when closing-or-static** (`relVel · n̂ ≤ 0`). If
    **separating**, do NOT merge — apply positional de-overlap only (mirrors the medium-separating
    path) so two bodies flying apart are never absorbed. This diverges intentionally from the
    prototype (which merges regardless of sign) to fix the snap-back.
  - MEDIUM (`V_MERGE ≤ vn < V_FRACTURE`): if **closing**, apply the restitution bounce + chip +
    dent + positional de-overlap; if **separating**, **no bounce impulse** (it would suck them
    together) but still apply positional de-overlap + a (smaller) deform/dent so overlapping
    bodies don't tunnel.
  - FAST (`vn ≥ V_FRACTURE`): fracture regardless of sign — the energy is already present. Fragments
    are spawned with a momentum-neutral burst (see fracture math) so they do not collectively snap
    back; and because SLOW now requires closing, any residual fragment overlap resolves as
    de-overlap, not re-merge.
  - A purely separating overlap that is neither merged nor bounced still returns a `regime` value
    (`'none'` with a de-overlap correction) — never `undefined`.
- **Contract-gap resolution (push-out + dents):** the frozen descriptor's survivor entry is
  **extended additively** (does not break the frozen shape, only adds optional fields):
  `survivors:[{ id, mNew, vx, vy, x?, y?, deformInjection?, dent? }]` where `x,y` carry the
  post-de-overlap position correction and `dent` is a `[{index, delta}]` spec the Workflow applies
  to `verts`. Document this extension in the descriptor's JSDoc as the resolution of the recon's
  flagged gap.
- **Gravity cap-vs-differentiation:** normalization makes the exponent change the *ratio* of
  pulls, not absolute scale, so the per-body `GRAV_ACC_CAP` stays a rare backstop rather than the
  active constraint. The exponential-differentiation invariant is tested as a **ratio below the
  cap regime** (test the un-capped force ratio), independent of final cap tuning. Note that
  `M_REF` **cancels in the ratio** (`pull(mBig)/pull(mSmall) = (mBig/mSmall)^GRAV_EXP`), so the
  ratio test does NOT exercise the normalization — a *separate* test pins normalization
  (DIVERGENCE 1) so dropping `/M_REF` is caught. Default `M_REF`, `GRAV_EXP`, softening, and cap
  are sensible starters; **final balancing is Sub-Plan 04**.

- **dt-scaling policy (explicit, per the GameLoop `dtNorm` convention):** `dtNorm` is per-frame
  at 60fps, clamped `[0,2]`. The prototype's constants fall into three classes — port each with
  the mathematically correct dt rule, NOT a blanket `*dtNorm`:
  - **Linear (`× dtNorm`)** — additive deltas: velocity integration `v += a*dtNorm`, position
    `pos += v*dtNorm`. (At `dtNorm=1` these reproduce the prototype exactly.)
  - **Exponential (`** dtNorm`)** — geometric per-frame decays: drag `0.999**dtNorm`, contact
    flash `0.86**dtNorm`, spring damping if dt-scaled. A blanket `*dtNorm` on these is WRONG.
  - **Frame-locked (dt-independent)** — the soft-body **spring** runs once per frame at a fixed
    cadence. Naively dt-scaling each spring term changes the damping ratio and can go unstable at
    `dtNorm=2`, so `softBodyStep` is **frame-locked**: its signature is `softBodyStep(body)` with
    **no `dtNorm` parameter** (the recon/earlier draft's `softBodyStep(body, dtNorm)` is corrected
    here to avoid a dead, misleading parameter). The spring is cosmetic; frame-locking is stable
    and acceptable. `flash` decay, being a simple geometric decay, MAY live in `softBodyStep` but
    uses the exponential rule (`flash * 0.86**dtNorm`) — if so, `softBodyStep(body, dtNorm)` keeps
    the param ONLY for the flash term and the spring stays frame-locked; pick one and document it.
    Recommended: keep `softBodyStep(body)` frame-locked including flash (simplest, stable), since
    at clamped `dtNorm≤2` the visual difference is negligible.
  - **Gravity accel cap** caps *acceleration* (`GRAV_ACC_CAP`); under a 2-frame stall the applied
    per-frame velocity delta is `a*dtNorm ≤ 2*cap`. This is the intended stall catch-up, documented
    so it is not mistaken for a cap violation.

## Module surface (all under `src/Operations/`, pure)

- **`vec.js`** — tiny helpers: `len(x,y)`, `normalize(x,y) -> {x,y}` (coincident → fallback axis),
  `dot(ax,ay,bx,by)`. (No reusable vector math exists today; `affine.js` is render-only.)
- **`gravity.js`** — `gravityAccel(bodies) -> Array<{ax,ay}>` (index-aligned with `bodies`).
  Softened N-body: for each ordered pair, `f = G * (mOther/M_REF)^GRAV_EXP / (d² + GRAV_SOFTEN)`
  toward the other body; coincident-center guard (`d < EPS` → skip/return 0, never divide by 0);
  `Number.isFinite` guard → 0; **per-body acceleration cap** clamps each body's *accumulated*
  acceleration magnitude to `GRAV_ACC_CAP`. Uses an optional `massCeiling` (clamp `m` to
  `MASS_CEILING` before normalization) so unbounded accretion can't blow up the super-linear term.
  Pure: reads positions/masses, returns accelerations.
- **`integrate.js`** — `integrate(body, accel, dtNorm, { drag, velCap }) -> {vx, vy, x, y}`:
  applies `v += a*dtNorm`, drag as `v *= drag**dtNorm`, clamps speed to `velCap`, then
  `pos += v*dtNorm`. **Policy-free** — it does NOT read `body.player`. The drifter-vs-player
  difference (drifters: `drag≈0.999, velCap=VEL_CAP`; player later: `drag=1, velCap=Infinity`
  + thrust) is the caller's concern, passed as params, keeping that volatility in the Workflow
  (Sub-Plan 03) where it belongs. Pure (returns new kinematics).
- **`softbody.js`** —
  - `applyDeform(body, axisX, axisY, magnitude) -> deformInjection {def, dax, day, dvel, flash}`:
    the collision-triggered kick (prototype lines 214–220) — normalized axis, `def = min(DEF_MAX,
    max(body.def, mag))`, `dvel -= mag*0.6`, `flash += mag*1.6`. Returns the injection; does not
    mutate.
  - `softBodyStep(body) -> {def, dvel, flash}`: one **frame-locked** damped-spring step (prototype
    lines 267–276) — `dvel += (0-def)*DEF_SPRING; dvel *= DEF_DAMP; def += dvel`, clamp `def` to
    `[0, DEF_MAX*1.2]`, `flash *= 0.86`. No `dtNorm` param (see dt-scaling policy — the spring is
    frame-locked for stability). Pure; caller assigns the returned state. NOTE: this is an
    **underdamped** spring — `def` overshoots and oscillates toward 0 ("jelly settle"), it does
    NOT decrease monotonically; only `flash` decays monotonically.
- **`growth.js`** — `absorbedMass(bigM, smallM) -> number` (diminishing-returns accretion curve,
  prototype lines 318–337: `smallM * ABSORB_BASE * max(DIMINISH_FLOOR, 1/(1+bigM/DIMINISH_SCALE))`)
  and `catchEfficiency(closingSpeed) -> number` (high-velocity catch loss,
  `1 - min(CATCH_SPEED_LOSS_MAX, CATCH_SPEED_LOSS_K * closingSpeed)`). Pure helpers used by both
  merge and size-mismatch-fracture paths.
- **`collision.js`** —
  - `overlapping(a, b) -> {hit, d, nx, ny, penetration, sumr}` narrow-phase test using
    `MASS_TO_R` radii (prototype lines 284–305), with the coincident-center jitter using `rng`.
  - `resolveCollision(a, b, budget, rng = Math.random) -> effectDescriptor`: the three-regime
    resolver. Returns the **full descriptor** (see below). Pure. `budget` = remaining body-count
    room (`MAX_BODIES - currentPopulation`), passed in; gates fracture fragment count and forces
    the particle-fallback when `< 2` real fragments fit.

### The effect descriptor (returned by `resolveCollision`)

```
{
  regime: 'merge' | 'chip' | 'fracture' | 'none',
  survivors: [ { id, mNew, vx, vy, x?, y?, deformInjection?, dent? } ],
  dead: [ id, ... ],
  fragments: [ Body, ... ],           // real bodies via makeBody(), momentum-inheriting
  counters: { eaten, merged, shatter }, // increments this collision contributes (0/1 each)
  fx: { debris: [ {x,y,vx,vy,hue,life} ], shock: [ {x,y,maxR,hue} ] }  // flavor data only
}
```

The Workflow (Sub-Plan 03) applies it: set survivor mass/velocity/position, apply
`deformInjection` and `dent`, remove `dead`, append `fragments`, add `counters` to globals, push
`fx` into the renderer's `particles`/`shocks`. Operations never touches any of that.

**Apply-semantics (specified so Sub-Plan 03 applies consistently — closes the recon/DA contract
gap):**
- `deformInjection = { dax, day, defTarget, dvelDelta, flashAdd }`. The Workflow applies:
  `body.def = max(body.def, defTarget)` (the prototype's `max` accretion of squash),
  `body.dvel += dvelDelta`, `body.flash = min(1, body.flash + flashAdd)`, `body.dax = dax;
  body.day = day`. (Absolute targets + documented combine rules — NOT blind assignment — so two
  injections in one frame compose like the prototype.)
- `dent = [{ index, value }]` with **absolute, already-clamped final** vertex values
  (Operation computes `value = max(0.55, verts[index] - reduction)` using the injected `rng`);
  the Workflow simply assigns `verts[index] = value`. No apply-time clamping needed, no divergence
  between implementers.
- `survivors[].x/y` are post-de-overlap absolute positions (omit when unchanged).
- `fragments` are **new `Body` objects** the Operation creates via `makeBody()` and may freely
  initialize (set `def/dax/day/vx/vy`) before returning — they are not inputs, so this does not
  violate input-purity.

### Regime math (ported from prototype, with the two divergences)

- **MERGE** (`vn < V_MERGE`): `newM = bigM + absorbedMass(bigM, smallM)`. Survivor velocity
  **`v = (bigM*bigV + smallM*smallV) / newM`** (DIVERGENCE 2 — exact momentum / new mass). Small
  body → `dead`. `counters.merged=1` for comparable sizes (ratio ≤ `SAME_SIZE`), else
  `counters.eaten=1` (big-eats-small). Both bodies get a `deformInjection` along the normal.
- **CHIP/BOUNCE** (`V_MERGE ≤ vn < V_FRACTURE`, closing): 1D normal restitution impulse
  `j = -(1+RESTITUTION) * vRelNormal / (1/mA + 1/mB)` distributed to both velocities; positional
  de-overlap split by inverse mass → survivor `x,y`; chip one real fragment off the larger if its
  mass `> CHIP_MIN_MASS` and `budget ≥ 1` (`chipM = CHIP_FRACTION*m`, fragment via `makeBody`
  inheriting position + a small `rng` spray velocity, parent `mNew -= chipM`); `dent` both
  silhouettes (2 adjacent verts inward, `rng`-chosen); `deformInjection` + `flash` both. Separating
  sub-path: de-overlap + reduced deform/dent, **no impulse**, no chip.
- **FRACTURE** (`vn ≥ V_FRACTURE`): parents = comparable ? `[a,b]` : `[smaller]` (size-mismatch:
  the much-bigger body instead grabs mass with `catchEfficiency` and dents — a violent catch).
  For each parent: fragment count `nf = clamp(2, MAX_FRAGS_PER_HIT, 2 + floor(intensity*2))` where
  `intensity = vn / IMPACT_REF_SPEED`, further clamped by `budget`; total surviving fragment mass
  `= parentM * (1 - FRAG_MASS_LOSS)` split equally across `nf` fragments (the `FRAG_MASS_LOSS` is
  the massless dust). **Momentum-neutral burst (corrected — the prototype's raw `rng` burst does
  NOT conserve momentum):** fragment velocities are `parentV + burst_k` where the `burst_k` are
  constructed to sum (mass-weighted) to **zero** — use evenly-spaced base angles
  `2πk/nf + rng-jitter`, build the burst vectors, then **subtract their mean** so
  `Σ burst_k = 0`. Then summed fragment momentum `= Σ (each * (parentV + burst_k)) = each*nf*parentV
  = (1 - FRAG_MASS_LOSS)*parentM*parentV` — i.e. **momentum of the surviving mass is conserved
  exactly**, with the lost `FRAG_MASS_LOSS` treated as massless dust carrying none. (This is the
  coherent invariant test #4 asserts; the original "equals the full parent momentum" claim was
  arithmetically impossible given the mass loss.) Parent → `dead`, `counters.shatter=1`. If
  `nf < 2` (no room): **particle fallback** — no real fragments, emit up to `MAX_DEBRIS_PER_HIT`
  `fx.debris` instead, parent survives dented (budget-safe, anti-freeze).

## Verification (TDD, invariant-first — `test/operations/*.test.js`)

Write tests asserting the **intended invariants**, not the prototype's arithmetic. Each test
must be shown to **fail when the behavior is inverted/broken** (temporarily break it, confirm red,
restore) — note the falsifiability check in a comment, per project rules.

1. **Gravity exponential-differentiation (ratio, not magnitude):** for two masses `mBig > mSmall`
   at equal distance, `pull(mBig)/pull(mSmall)` **strictly increases as `GRAV_EXP` increases**
   (compute at EXP=1.0, 1.6, 2.2). Holds for any `M_REF` placement (it cancels:
   ratio = `(mBig/mSmall)^EXP`). Falsify: with `GRAV_EXP=1` the ratio is constant — assert the
   strict-increase test fails there.
1b. **Gravity normalization pin (DIVERGENCE 1 — since `M_REF` cancels in test 1, this guards it):**
   assert absolute pull magnitude at a known mass/distance equals
   `G*(m/M_REF)^GRAV_EXP/(d²+GRAV_SOFTEN)`, AND that scaling **all** masses and `M_REF` by the same
   factor leaves the acceleration field invariant. Falsify by dropping the `/M_REF` (or changing
   only `M_REF`) → the absolute-pull / invariance assertion fails. (Without this, removing the
   normalization is silently green.)
2. **Gravity cap + NaN guards:** accumulated per-body acceleration magnitude never exceeds
   `GRAV_ACC_CAP`; coincident centers (`d=0`) return finite `0`, never NaN/Infinity; a body never
   self-attracts. Falsify by removing the guard → NaN appears.
3. **Slow-merge momentum exact + mass loss (DIVERGENCE 2):** total momentum after merge equals
   `bigM*bigV + smallM*smallV` to tolerance (survivor carries it via `combinedMomentum/newMass`),
   while `newMass < bigM+smallM` by exactly the diminishing-returns debris loss. Falsify by
   dividing by `combinedMass` → momentum test fails. (Also assert the slow-**separating** case does
   NOT merge, per the regime table.)
4. **Fracture conservation (corrected invariant):** summed fragment **mass** =
   `parentM*(1-FRAG_MASS_LOSS)` to tolerance; AND summed fragment **momentum** =
   `(1-FRAG_MASS_LOSS)*parentM*parentV` to tolerance (the surviving mass's momentum — the
   momentum-neutral burst sums to zero, the lost dust carries none). Falsify two ways: (a) skip the
   mean-subtraction so `Σ burst ≠ 0` → momentum assertion fails; (b) change the mass split → mass
   assertion fails. (Do NOT assert momentum equals the *full* `parentM*parentV` — that is
   arithmetically impossible under mass loss.)
5. **Regime selection matrix:** across the full grid of `vn ∈ {below V_MERGE, exactly V_MERGE,
   between, exactly V_FRACTURE, above}` **crossed with closing vs separating** normal sign, assert
   the chosen `regime` and whether a bounce impulse fired matches the decided table (boundaries
   inclusive per `<`/`≥`), including the explicit cells: SLOW+closing→merge, SLOW+separating→none
   (de-overlap, no merge), MEDIUM+closing→chip+bounce, MEDIUM+separating→none (de-overlap, no
   impulse), FAST→fracture (either sign). Every cell returns a defined `regime`, never `undefined`.
   Falsify by flipping a boundary `<` to `≤`, or by removing the SLOW closing-sign guard (then
   SLOW+separating wrongly merges) → the matrix fails.
6. **Budget gating / particle fallback:** with `budget = 0` a fast fracture produces **zero real
   fragments** and emits `fx.debris` instead, parent survives dented; with ample budget it
   produces `2..MAX_FRAGS_PER_HIT` real bodies clamped by `budget`. Falsify by ignoring `budget`
   → fragments exceed the cap.
7. **Soft-body spring (stability + flash decay):** `softBodyStep` is a stable underdamped spring —
   `def` is **bounded** (never exceeds `DEF_MAX*1.2`) and **converges to ~0** over many steps (it
   may overshoot/oscillate — do NOT assert monotonic decrease); `flash` decays **monotonically**
   (`*0.86`). `applyDeform` returns an injection with normalized axis and `defTarget ≤ DEF_MAX`.
   Falsify by removing damping (`DEF_DAMP=1`) → `def` diverges past the bound.
8. **dt-scaling policy:** at `dtNorm=2`, drag and flash decays match the **exponential** form
   (`0.999**2`, `0.86**2`), not the linear `*2` form; `integrate` velocity delta equals
   `a*dtNorm`. Falsify by using `*dtNorm` on the decays → the dtNorm=2 assertion fails.
9. **Determinism:** with a seeded `rng` and `_resetIdCounter()`, two identical `resolveCollision`
   calls return byte-identical fragment ids, positions, and dents.
10. **Purity:** `resolveCollision`/`gravityAccel`/`softBodyStep`/`integrate` do **not** mutate
   their input bodies (deep-freeze the inputs; the call must not throw and must return new data).
   (Creating + initializing new fragment Bodies inside `resolveCollision` is allowed — they are
   not inputs.)

Run: `npx vitest run` — the existing 20 tests (affine, body) plus these must all pass. Keep the
existing tests green (no edits to the frozen `Body` contract or render math).
