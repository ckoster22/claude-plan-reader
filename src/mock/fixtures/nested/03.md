# Sub-Plan 03: Input, Controls & Infinite-World Population Management

> Part of the **Chompy Asteroids** master plan. This sub-plan is the **integration point**: it
> wires the real per-frame simulation into `GameLoop`'s `step` seam, adds player input/controls,
> and builds the infinite-world population manager. It consumes Sub-Plan 02's pure physics and
> Sub-Plan 01's render spine. Physics math, render internals beyond the FX sink, and final HUD
> polish/tuning are non-goals (Sub-Plan 04).

## Context

Sub-Plans 01–02 delivered: the render spine + frozen `Body` contract + dt-normalized `GameLoop`
(`startGameLoop({step, render, now})`, `clampDtNorm`), and the **pure** physics library under
`src/Operations/` (`gravityAccel(bodies)`, `integrate(body, accel, dtNorm, {drag, velCap})`,
`softBodyStep(body)` [frame-locked], `applyDeform`, `overlapping(a,b,rng)`,
`resolveCollision(a,b,budget,rng)`, growth helpers). `main.js` currently runs a **no-op `step`**
and a mock-driven render (`mockDrift → renderer.tick → starfield.draw → renderer.draw →
hud.update`). `src/Connectors/PixiRenderer.js` holds empty `particles`/`shocks` arrays and a
cosmetic `tick()` spin.

This sub-plan replaces the mock with a real simulation and an infinite, self-populating world,
porting the mechanics from the prototype's `step()` (lines 222–449), spawn helpers (132–188),
and pools (191–210).

**Parent adjustment (honored):** when wiring the real sim, **remove** the renderer's cosmetic
`rot += vr*dtNorm` (`PixiRenderer.tick`) and `mockDrift()` and stop calling them, so motion is
never double-applied; all player+drifter motion (incl. the `rot += vr*dtNorm` spin) flows through
the sim; pass `budget = MAX_BODIES − population` to `resolveCollision`; apply descriptors via the
deferred-fragment buffer using the documented apply-semantics.

  Replaces the prototype's module-global `keys`.

## WorldManager (`src/Workflows/WorldManager.js`)

State it owns: `bodies` (array), `particles`, `shocks`, `counters {eaten,merged,shatter}`, and an
injectable `rng` (default `Math.random`). Helpers (all bounded / NaN-safe, ported):
- **`viewRect(cam, W, H)`** — `{x0,y0,x1,y1,cx,cy,vw,vh}` from `cam.x/y/z` (prototype 132–135).
  Top-left anchored to match the renderer's camera. (May live here or a tiny shared util.)
- **`powerMass(rng)`** — `MASS_MIN + rng()**POWER_EXP * MASS_SPAN` (small-weighted, 113–118).
- **`offscreenPos(viewRect, rng)`** — pick a point in the ring beyond the viewport on a random
  side (138–148). Pure given `viewRect`+`rng`.
- **`spawnDrifter(viewRect, playerPos, rng)`** — **bounded** safe-spawn loop (`≤40` tries) that
  guarantees a fallback position if no try clears `SAFE_R` of the player; inward drift velocity
  (155–168). Returns a `makeBody`.
- **`spawn()`** — seed: a player `makeBody({player:true, m:PLAYER_MASS, x:0, y:0})` at origin,
  then `POP` drifters with the `SEED_SMALL_GUARANTEE` small-body guarantee (175–188). Player is
  index-tracked so it's never culled.
- **`cullAndRespawn(viewRect, rng)`** — filter out non-player bodies beyond `CULL_MARGIN` of the
  viewport (player exempt, 429–433), then top up with `spawnDrifter` until `POP` (or `MAX_BODIES`),
  with a `guard` counter bounding the loop (435–437).
- **`applyThrust(player, inputState, dtNorm, pushParticle)`** — pure acceleration, **no drag**:
  `vx += THRUST*dtNorm * dirX` etc. from `inputState`; **clamp speed to `PLAYER_VEL_CAP` HERE
  (thrust-driven cap only)**; if thrusting, emit **exactly one** exhaust particle per thrusting
  frame (discrete event — NOT scaled by dtNorm) opposite the thrust direction (236–239) via
  `pushParticle`. No-ops safely if `player` is undefined/dead. (Player coasting = momentum
  conserved on release; the player is integrated with `drag=1`.)
  **Player velocity-cap decision (prototype-faithful, resolves the double-clamp):** the prototype
  caps ONLY thrust-driven speed (line 241–242) and lets **gravity accelerate the player with no
  cap** (the drifter `VEL_CAP` clamp skips the player, line 261). Therefore the player is
  integrated with **`velCap: Infinity`** (NOT `PLAYER_VEL_CAP`) so `integrate`'s clamp never
  fires for the player — the only player cap is this `applyThrust` clamp. Drifters use
  `velCap: VEL_CAP`. This avoids the double-clamp and preserves gravity slingshots.
- FX pools: **`pushParticle(p)`** (drop-oldest at `MAX_PARTICLES`, 191–194), **`debrisBurst(...)`**
  (≤`MAX_DEBRIS_PER_HIT`, 197–206), **`addShock(...)`** (≤`MAX_SHOCKS`, 207–210),
  **`advanceFx()`** — **frame-locked** (no `dtNorm`, matching `softBodyStep`'s precedent: these
  are cosmetic, and half-scaling position-but-not-life is the worst option). Port the prototype
  per-frame math verbatim: particles `x+=vx; y+=vy; vx*=PARTICLE_DRAG; vy*=PARTICLE_DRAG; life--`;
  shocks `r += (max-r)*0.18; life -= 0.06`; filter dead (439–442).

## SimStep (`src/Workflows/SimStep.js`) — per-frame order (bodies-array safety is critical)

`makeSimStep(world, input) -> step(dtNorm)` where `world` is the WorldManager and `input` exposes
`inputState`. Each frame, **in this exact order**:
1. `const me = world.bodies.find(b => b.player)`.
2. **Thrust:** if `me`, `world.applyThrust(me, input.inputState, dtNorm, world.pushParticle)`.
3. **Gravity:** `const acc = gravityAccel(world.bodies)` — capture the **index-aligned** accel
   array BEFORE any position mutates (recon risk #2).
4. **Integrate + spin + soft-body**, per body `i` (positions only change here, after accel is
   captured): `integrate(b, acc[i], dtNorm, b.player ? {drag:1, velCap:Infinity} :
   {drag:DRIFTER_DRAG, velCap:VEL_CAP})` → assign returned `vx,vy,x,y` (player `velCap:Infinity`
   per the cap decision above — gravity is uncapped for the player); `b.rot += b.vr*dtNorm`
   (the spin relocated from the deleted `renderer.tick`); apply `softBodyStep(b)` → assign
   `def,dvel,flash`.
5. **Collisions:** double loop `i<j` over `world.bodies`. **FIRST, per pair, the dead-guard:**
   `if (deadIds.has(a.id) || deadIds.has(b.id)) continue;` — a body merged/fractured away by an
   earlier pair this frame must NOT collide again (the prototype's line-287 guard;
   `collision.js` is pure and will NOT self-guard, so this MUST live here — omitting it
   **duplicates mass** in dense clusters, the single most likely correctness bug). Then
   `const info = overlapping(a,b,rng)`; **only if `info.hit`** call
   `const d = resolveCollision(a,b,budget,rng)` with
   `budget = MAX_BODIES - (world.bodies.length + newFrags.length)` (recompute per pair as frags
   accumulate). **Apply `d`** (apply-semantics below) — survivors/dead/counters/fx applied
   immediately, but `fragments` are pushed to a **deferred `newFrags` buffer**, never into
   `world.bodies` mid-iteration. Add each `dead` id to `deadIds`.
6. **Commit:** `world.bodies = world.bodies.filter(b => !deadIds.has(b.id))`; then append
   `newFrags` while `world.bodies.length < MAX_BODIES` (424).
7. **Cull/respawn:** **re-resolve the player AFTER commit** (the step-1 `me` may now be dead/merged
   away — do NOT reuse it): `const mePost = world.bodies.find(b=>b.player)`; derive
   `playerPos = mePost ? {x:mePost.x, y:mePost.y} : {x:vr.cx, y:vr.cy}` (viewport-center fallback,
   prototype line 157). `world.cullAndRespawn(vr, rng)` where `vr = viewRect(cam, W, H)`; the
   safe-spawn `SAFE_R` check uses `playerPos`. If `W ≤ 0 || H ≤ 0` (degenerate first frame), skip
   cull/respawn this frame (anti-degenerate guard).
8. **FX advance:** `world.advanceFx(dtNorm)`.
9. Camera follow stays in `renderer.draw` (already player-driven via `updateCamera`); no change.

### Descriptor apply-semantics (from Sub-Plan 02's documented contract)
For each survivor (matched to a live body by `id`): `m = mNew`; `vx,vy` set; if `x,y` present set
them; if `deformInjection` present: `def = max(def, defTarget); dvel += dvelDelta; flash =
min(1, flash+flashAdd); dax = dax; day = day`; if `dent` present: `verts[index] = value` for each.
`dead[]` → mark for removal. `fragments[]` → deferred buffer. `counters` → add to
`world.counters`. `fx.debris` → `pushParticle` each; `fx.shock` → `addShock` each.

## main.js wiring (the integration)

- **W/H single-sourcing + non-zero spawn:** `viewRect` (in the step) and `updateCamera` (in the
  renderer) must read the **same** `W,H`. `PixiRenderer.updateCamera` currently reads
  `app.renderer.width/height` directly while `main.js` caches its own `W,H` — unify them (pass the
  closure `W,H` into `updateCamera`, or have both read `app.renderer.width/height`). Verify the
  canvas has non-zero size after `await app.init({resizeTo:window})` before `world.spawn()` (the
  off-screen spawn ring degenerates if `W/H===0`); if needed, defer `spawn()` to the next frame /
  after a size check so the initial POP isn't clustered at the origin inside `SAFE_R`.
- `const world = new WorldManager(rng); /* after non-zero W,H confirmed */ world.spawn();`
- `const input = new Input(window);`
- `const step = makeSimStep(world, input, () => viewRect(cam, W, H));` (the `viewRect` thunk reads
  the live closure `W,H`, refreshed by the resize handler).
- `const render = (dtNorm) => { starfield.draw(cam, W, H); renderer.draw(world.bodies, cam,
  { particles: world.particles, shocks: world.shocks }); hud.update(buildHudVM(world)); };`
  (No `mockDrift`, no `renderer.tick`.)
- `buildHudVM(world)` reads `world.counters` (use `?? 0` so a missing field never shows NaN) for
  eaten/merged/shatter and the live `world.bodies` for total/biggest/mass/rank/growth (the player
  may be `dead`/absent → VM `mass:null` triggers the dormant game-over overlay; full lose/restart
  is Sub-Plan 04, but the VM must not throw on an empty array or absent player).
- `startGameLoop({ step, render })`. Keep `window.__chompy = { ..., world }` for verification.

## Verification

**Unit tests (`test/workflows/*.test.js`, Vitest, deterministic via injected `rng`/`viewRect`):**
1. **Off-screen-only spawn:** `spawnDrifter`/`offscreenPos` for a given `viewRect` always returns a
   point OUTSIDE `[x0,x1]×[y0,y1]` (ring beyond the margin), for many seeds. Falsify: place inside
   the viewport → test fails.
2. **Safe-spawn distance:** every spawned drifter is `≥ SAFE_R` from the player when a valid
   placement exists; the bounded loop (`≤40` tries) always returns (never hangs) and uses the
   guaranteed fallback when crowded. Falsify: remove the SAFE_R check → a spawn lands within SAFE_R.
3. **Power-law small-weighting:** over N=10k `powerMass(rng)` draws, the median is small (well
   below mid-range) and large bodies are rare (e.g. >50% below `MASS_MIN + 0.05*MASS_SPAN`).
   Falsify: replace with uniform → the distribution test fails.
4. **Cull keeps player + bounds density:** `cullAndRespawn` never removes the player even when far
   off-screen; after culling everything and respawning, population returns to ~`POP` and never
   exceeds `MAX_BODIES`. Falsify: cull the player → test fails.
5. **Anti-freeze caps:** drive many collisions/spawns; assert `bodies.length ≤ MAX_BODIES`,
   `particles.length ≤ MAX_PARTICLES` (drop-oldest), `shocks.length ≤ MAX_SHOCKS`; the spawn loop
   `guard` bounds iterations. Falsify: remove a cap → length exceeds it.
6. **Deferred-buffer integrity (falsifiable — frags don't collide the frame they're born):** seed
   a `SimStep` frame with a fast comparable pair that fractures into ≥2 fragments positioned so a
   fragment overlaps an original body. Assert that in **this** frame NO fragment-vs-original
   collision is resolved (frags enter `bodies` only at commit, after the pair loop) — e.g. the
   frame's `shatter`/`merged` counts equal the original-pairs-only outcome, and total mass matches
   the single-fracture expectation. Falsify: push frags into `bodies` *inside* the loop → a
   frag-vs-body collision fires the same frame and the counter/mass assertion changes. (Pure
   pre/post length checks can't distinguish deferral, so this asserts the observable consequence.)
6b. **No mass duplication from the per-pair dead-guard:** seed three mutually-overlapping bodies
   A,B,C (slow, closing) where A would merge B in pair (A,B) and then be revisited in pair (A,C).
   Assert A's final mass reflects each body absorbed **at most once** (no corpse re-merge) and
   total field mass is conserved-minus-loss, not inflated. Falsify: remove the
   `deadIds.has(a.id)||deadIds.has(b.id)` guard → a dead body collides again and mass inflates.
7. **Descriptor application asserts the documented INVARIANT (not reverse-engineered from code):**
   feed `SimStep` a **hand-built** descriptor with known survivor `deformInjection`/`dent` and
   assert the Workflow applies the combine RULES from the contract: `def === max(prevDef,
   defTarget)` (NOT `def === defTarget`), `flash === min(1, prevFlash+flashAdd)`,
   `dvel === prevDvel + dvelDelta`, `verts[index] === value`, `dead` removed, `counters` summed,
   `fx` pushed. Do NOT assert which body chips or copy prototype line numbers — assert the combine
   rule. Falsify: change the applier to `def = defTarget` (drop the `max`) → the test fails.
8. **Coasting / no-drag player vs. drifter drag:** after `applyThrust` then several no-input
   integrate steps, the player's speed is unchanged (drag=1, below cap); a drifter's speed decays
   (`drag=DRIFTER_DRAG`). Falsify: apply drag to the player → speed decays.
9. **Single-sourced motion:** assert `PixiRenderer` no longer exports/uses `tick` and `main.js`
   does not import `mockDrift`; the sim advances `rot` (a body's `rot` changes only via `SimStep`,
   confirmed by stepping with the renderer absent).

Each invariant test shown red-then-green (break it, confirm failure, restore) per project rules.
Keep the existing 59 tests green. Run `npx vitest run` and `npm run build`.
