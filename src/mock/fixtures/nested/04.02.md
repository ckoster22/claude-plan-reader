# Sub-Plan 04.02: HUD Surfacing & VM-Mapping Tests

> Leaf child of Sub-Plan 04. Builds on the committed green baseline (HEAD `d6846a1`). Scope:
> make the HUD view-model a **pure, unit-testable** module and add **falsifiable** VM-mapping
> tests. The optional debug overlay is **CUT** (parent adjustment). Frozen layers
> (`src/Operations/*`, `src/domain/Body.js`, `src/math/affine.js`) are READ-ONLY.

## Context

The live HUD already works: `buildHudVM()` (an inline closure in `main()` at `src/main.js:85–108`)
reads `world.bodies` + `world.counters` every frame and feeds `Hud.update` (`src/Connectors/Hud.js`),
which writes `#mass/#rank/#total/#biggest/#growth/#merged/#eaten/#shatter` and toggles `#over` on
`mass==null`. Recon confirmed **no stale/zero/mis-computed field** vs the prototype reference
(`growthPct = me.m/biggest*100`, `rank = mass-descending index+1`, "size" = mass `me.m|0` — all
correct). So the mandate's "fix any field that reads stale/zero" is a no-op precaution here.

The real gap is **testability**: `buildHudVM` is a closure inside `main()`, and `main.js` imports
Pixi `Application` + does DOM bootstrap, so it can't be imported headlessly in Vitest. 04.02's
substantive deliverable is therefore: **extract the mapping to a pure module and pin it with
falsifiable unit tests** — locking the HUD contract before 04.03 widens the VM (game-over copy)
and 04.04 tunes constants.

## Changes

1. **New pure module `src/Workflows/hudVM.js`** exporting `buildHudVM(world)` — a verbatim move of
   the current logic (`main.js:85–108`), parameterized on `world` instead of closing over it. The
   signature is `buildHudVM(world)` (reads `world.bodies` / `world.counters` — NOT a destructured
   `{bodies,counters}` param). It returns the VM `{ total, biggest, eaten, merged, shatter, mass,
   rank, growthPct }`, and imports NOTHING from DOM/Pixi (pure data → data). Preserve every current
   behavior exactly: `biggest = '—'` when empty; counters `?? 0`-guarded; player branch sets
   `mass: me.m|0`, `rank: indexOf+1`, `growthPct: top>0 ? min(100, me.m/top*100) : 0`; no-player
   branch returns `mass: null, rank: '—', growthPct: 0`. (Keeping the `'—'`/`null` sentinels
   identical matters — `Hud.js` and the `#over` toggle depend on `mass==null`.)
2. **`src/main.js`** — import `buildHudVM` from `./Workflows/hudVM.js`, delete the inline closure,
   and change the render call site to `hud.update(buildHudVM(world))`. That is the ONLY production
   edit; the committed physics/SimStep/WorldManager baseline is untouched.
3. **DO NOT touch** `src/Connectors/Hud.js` (specifically the `#otitle` omission and the hard-coded
   `#osub` string at `Hud.js:49–54`) — those are 04.03's game-over-copy work. Do not touch
   `index.html`, `world.counters` reset behavior, or any frozen layer.
4. **Debug overlay: CUT** (parent adjustment). Do not add a Pixi Graphics layer, a `~` toggle in
   `Input.js`, or thread `SimStep`'s `acc` out — it would perturb the committed baseline and has no
   headless verification path (chrome-devtools unavailable). Record the cut explicitly so 04.04's
   end-to-end step doesn't expect it.

## Verification — falsifiable VM-mapping unit tests

New `test/workflows/hudVM.test.js` (Vitest), constructing plain worlds via `makeBody(opts)`
(`src/domain/Body.js`) + literal `counters` objects — the established `test/workflows/*` pattern
(no `main.js`/Pixi import). Each assertion paired with a red-then-green falsifiability check (break
the mapping, confirm the test fails, restore — note it in a comment). Cover:
- **rank:** player among several bodies of differing mass → `rank` = its 1-based mass-descending
  position (e.g. player 3rd-biggest → `rank===3`). Falsify: swap to ascending → wrong rank.
- **growthPct (assert the formula, not the unreachable clamp):** player not-biggest →
  `growthPct === me.m/top*100` exactly (a real load-bearing invariant); player IS the biggest →
  `growthPct === 100`; `top>0` guard → no divide-by-zero on a degenerate zero-mass world.
  **Do NOT** claim "drop the `min(100,…)` clamp → exceeds 100" as the falsify — that is
  **unreachable**: the player is always a member of the sorted set so `me.m ≤ top` always, making
  the clamp dead for this contract and the test unfalsifiable. Instead make the falsify target the
  reachable formula/guard: swap the denominator (e.g. `me.m/total` or a fixed constant) → the exact
  `me.m/top*100` assertion fails.
- **counters:** `eaten/merged/shatter` pass through from `world.counters`; a **missing** counter key
  → `0` (the `?? 0` guard). Falsify: remove `?? 0` → `undefined` leaks.
- **total / biggest:** `total === bodies.length`; `biggest === max mass | 0`; **empty world** →
  `biggest === '—'`, `total === 0`. Falsify: use `sortedAsc[0]` → wrong biggest.
- **no-player (lose path):** a world with no `player:true` body → `mass === null`, `rank === '—'`,
  `growthPct === 0` (this is the `#over` trigger 04.03 builds on). Falsify: return `mass:0` → the
  `==null` overlay toggle would break.
- **purity:** `buildHudVM(world)` does not mutate `world.bodies`/`world.counters` (deep-freeze
  inputs; call must not throw — it only reads, spreads a copy, and sorts the copy). Falsify:
  replace `[...bodies].sort(...)` with an in-place `bodies.sort(...)` → mutating the frozen array
  throws in strict mode → red.

Then:
- `npx vitest run` → all green (existing 86 + the new hudVM cases; the moved logic must not change
  any existing behavior).
- `npm run build` → success (confirms the `main.js` import rewire is valid).
- Confirm no diff to `src/Operations/*`, `src/domain/Body.js`, `src/math/affine.js`,
  `src/Connectors/Hud.js`, `index.html`.
